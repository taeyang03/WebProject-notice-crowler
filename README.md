# LangGraph 기반 대학 공지사항 RAG 및 관리 시스템

본 프로젝트는 LangGraph와 FastAPI를 활용하여 대학 공지사항을 효율적으로 검색하고 요약하여 제공하는 RAG(Retrieval-Augmented Generation) 시스템입니다. 기간이 만료되었거나 일정 기간(180일)이 경과한 데이터는 검색 대상에서 제외하여 정보의 최신성과 신뢰성을 유지합니다.

---

## 1. 환경 설정 (Environment Setup)

본 시스템을 구동하기 위해 필요한 의존성 패키지 설치 및 환경 변수 설정 안내입니다. Python 3.10 이상의 환경을 권장합니다.

### 필수 패키지 설치
터미널에서 아래 명령어를 실행하여 핵심 라이브러리를 설치합니다.

```bash
pip install fastapi uvicorn pydantic dotenv slowapi
pip install langchain langchain-core langchain-openai langchain-chroma langgraph

```

### 환경 변수 설정 (`.env`)

프로젝트의 루트 디렉토리에 `.env` 파일을 생성하고 아래와 같은 형식으로 필수 키 값을 입력합니다.

```env
# OpenAI API Key (텍스트 임베딩 및 답변 생성에 사용)
OPENAI_API_KEY=sk-proj-...

# LangChain 내부 User-Agent 설정
USER_AGENT=notice-rag-agent/1.0

```

### 디렉토리 구조

서버 구동 전, 데이터베이스 엔진 및 웹 자원이 위치할 폴더 구조를 확인하십시오.

* `chroma_db/` : 공지사항 및 유저 구독 정보 데이터가 저장되는 Vector DB 경로입니다. (최초 실행 시 자동 생성)
* `public/` : UI 화면을 구성하는 HTML, CSS, JavaScript 등 정적 파일이 위치하는 경로입니다.

## 2. 사용법 (Usage)

### 서버 실행

환경 설정이 완료되면 아래 명령어를 통해 백엔드 서버를 구동합니다.


## 3. 주요 기능 (Key Features)

### 1. 데이터 최신성 필터링 (Recency-Aware Post Filtering)

* **마감일 분석:** 데이터의 유사도가 높더라도 메타데이터상의 마감일(`end_date`)이 현재 날짜보다 과거일 경우 답변 생성 대상에서 제외합니다.
* **기간 초과 제한:** 작성일(`date`) 기준 180일을 초과한 과거 공지사항은 포스트 필터링 로직을 통해 자동으로 제거되므로, 신뢰성 높은 정보만을 제공합니다.

### 2. 요청 수량 보장 시스템 (Dynamic Over-Sampling)

* 사용자가 "최신 공지 10개 요약해줘"와 같이 구체적인 수량을 지정하는 경우, 의도 분석 노드가 숫자를 인식하여 처리합니다.
* 필터링 단계에서 문서가 누락될 가능성을 고려하여, 요청 수량(`limit`)의 3배수 이상(`max(25, limit * 3)`)의 문서를 Vector DB에서 먼저 추출하는 동적 오버샘플링을 적용했습니다.

### 3. LangGraph 기반 의도 라우팅

* **워크플로우 제어:** 유저의 입력 맥락을 분석하여 `일반 대화(general)`, `학사 검색(search)`, `최신 요약(summary)` 노드로 명확하게 분기합니다.
* **개인화 검색 융합:** 사용자 로그인 상태인 경우, 유저가 사전 등록한 구독 키워드를 검색 쿼리에 논리적으로 결합하여 맞춤형 결과를 도출합니다.

### 4. 시스템 안정성 및 피드백

* **처리량 제한 (Rate Limiting):** 과도한 API 호출 및 트래픽 집중을 방지하기 위해 Slowapi를 통한 분당 최대 30회 제한이 걸려 있습니다.
* **스트리밍 아키텍처:** 사용자 경험을 향상시키기 위해 GPT의 답변 데이터를 실시간으로 전달하는 비동기 `StreamingResponse` 방식을 적용했습니다.
* **사용자 피드백 수집:** 답변별 긍정/부정 평가는 `feedbacks.jsonl` 파일에 적재되어 향후 RAG 품질 개선을 위한 로그 데이터로 활용됩니다.
